# AGENTS.md — CableCheck

Guidance for AI coding agents (Claude Code, OpenAI Codex CLI, and any subagents they
spawn) working in this repository. Human-facing usage lives in [README.md](README.md).
Design detail lives in [docs/](docs/). This file is the operational contract for *changing*
the code.

CableCheck is a **Go 1.26+, standard-library-only** Linux CLI. It tests the health of a
directly-connected Ethernet link between two PCs (PC1 coordinates, PC2 works) and emits a
rule-based verdict plus JSON / Markdown / text reports. Linux `amd64` and `arm64` only.

---

## Non-negotiables (read first)

1. **Standard library only.** `go.mod` has zero `require`s and there is no `go.sum`. Do not
   add third-party dependencies. `go mod tidy -diff` must stay clean.
2. **`gofmt` + `go vet` clean, always.** Run `make fmt` before finishing. Unformatted code
   fails `make fmt-check`.
3. **Tests stay hermetic.** No real `ping`/`ip`/`ethtool`/`iperf3`, no network, no real PIDs,
   no wall-clock sleeps, no `PATH` manipulation in test code. See [Testing](#testing).
4. **Never leak the session token.** It must not appear in any log, report field, or error
   string. See [Security invariants](#security--process-invariants).
5. **No AI attribution, anywhere.** No `Co-Authored-By`, no "generated by", no AI traces in
   commits, code comments, or docs. Claude owns git history — see [Git & commits](#git--commits).
6. **Verify before claiming done.** Run the gate yourself (`make check`). Don't trust a
   subagent's self-reported "tests pass". Race/shuffle hangs hide behind linear-order passes.

## Commands

The `Makefile` is the source of truth for *what* runs; GitHub Actions in `.github/workflows/`
just invoke these targets. `pr.yml`/`main.yml` run `make check` via the reusable `gate.yml`;
`security.yml` runs `govulncheck` + CodeQL; `release.yml` is a manual (`workflow_dispatch`)
build+publish. All builds use `CGO_ENABLED=0` for a static, dependency-free binary.

| Command | Purpose |
|---|---|
| `make build` | Build the static `cablecheck` binary into the repo root. |
| `make test` | `go test ./...` — fast, hermetic unit + integration tests. |
| `make test-race` | `go test -race -shuffle=on ./...` — **mandatory gate**; catches data races and test-order coupling. |
| `make fmt` | `gofmt -w .` — format in place. |
| `make fmt-check` | Fail if any file is unformatted. |
| `make vet` | `go vet ./...`. |
| `make lint` | `vet` + `fmt-check` + `staticcheck` **if installed** (optional; silently skipped when absent). |
| `make tidy-check` | `go mod tidy -diff` — module must stay stdlib-only. |
| `make check` | **Pre-commit gate:** `fmt-check` + `vet` + `tidy-check` + `test` + `test-race` + `build`. |
| `make dist` | Cross-compile `linux/amd64` + `linux/arm64` into `dist/`. |
| `make examples` | Regenerate `examples/` via `go run ./tools/genexamples`. |
| `make demo-e2e` | Build, then run two coordinated peers over loopback with `testdata/stubtools/` and validate report artifacts + transfer. |
| `make verify` | Full release gate: `check` + `dist` + `examples` + `demo-e2e`. |

**Before committing, run `make check`.** After changing report *output* format, also run
`go test ./internal/reporting -update` to regenerate golden files, then review the diff.

## Repository layout

Composition flows one way: `model` ← everything else ← `app` ← `cli` ← `cmd/cablecheck`.
`parser` and `evaluate` are pure transforms. `peer` knows the wire protocol but not health
rules. `app` is the composition root.

| Path | Responsibility |
|---|---|
| `cmd/cablecheck` | Entry point: signal handling, build-metadata injection (`main.version/commit/date` via ldflags). |
| `internal/cli` | Subcommand dispatch (`run`, `doctor`, `report`, `version`), flag parsing, exit-code mapping. Uses `flag.ContinueOnError` — never `ExitOnError`. |
| `internal/app` | Orchestration: preflight, report dir + logging, suite construction, peer session, evaluation/rendering, `doctor`/`report` commands, exit-code contract (`exit.go`). |
| `internal/peer` | TCP transport, authenticated handshake, session state machine, RPC correlation, abort/shutdown ordering, report-transfer routing. |
| `internal/protocol` | Framed JSON (4-byte length prefix + envelope), fixed message catalog, msg-ID dedup/ordering, constant-time token comparison. |
| `internal/config` | Flag parsing (fail-fast order), mode presets (quick/standard/soak), bounds validation, parameter resolution. |
| `internal/testsuite` | Test operations + per-mode coordinator plans; `RemoteCaller` RPC vocabulary for PC1/PC2. |
| `internal/evaluate` | Health-rule engine: pure fact extraction → ordered rule fold → classification + 0–100 score + findings. |
| `internal/parser` | Pure output parsers (`ethtool`, `ip -j`, `ping`, `iperf3`, cable-test/TDR). Bytes in, model out, no I/O. |
| `internal/runner` | Shell-free process execution: `exec.CommandContext` + `Setpgid`, SIGTERM→SIGKILL escalation, bounded capture, `LC_ALL=C`. |
| `internal/network` | Interface discovery/classification by owning IP, sysfs link monitoring, USB/virtual detection, port probes. |
| `internal/reporting` | Report dir layout + JSON/Markdown/text rendering + verified SHA-256 transfer + offline regeneration (`model` + stdlib only). |
| `internal/model` | Pure domain types: `Report`, results, `Bitrate`/`Duration` value objects, `HealthClass`. Imports stdlib only. |
| `internal/clock` | Injectable wall clock; `FakeClock` for tests. |
| `internal/logging` | `slog` multi-handler (human text → stderr, JSON debug → report dir) with token/payload redaction. |
| `internal/testutil`, `internal/runner/runnertest` | Hermetic test infrastructure (leak checks, `FakeRunner`, scripted stdin, dribble reader). |
| `testdata/` | Fixtures (`ip/ethtool/ping/iperf`), `golden/` (byte-exact report output), `stubtools/` (demo-only shell stubs — **tests never use these**). |
| `docs/` | `architecture.md`, `protocol.md`, `health-rules.md`, `report-schema.md`. |

## Code conventions

- **Errors:** wrap with context, e.g. `fmt.Errorf("pkg: doing X: %w", err)`. Discriminate with
  `errors.Is`/`errors.As`. Never return bare `err` at a boundary or `fmt.Errorf("%v", err)`.
- **Context:** every long-lived operation takes `ctx` as its first parameter. Time budgets via
  `context.WithTimeout`, cancellation via `context.WithCancel`, never hardcoded sleeps.
- **Subprocesses:** `exec.CommandContext` **only** (never bare `exec.Command`), always with
  `Setpgid=true`, and always append `LC_ALL=C` / `LANG=C`. All process spawning goes through
  `internal/runner`.
- **Time:** no `time.Now()` outside `internal/clock` and `main`. Injected `Clock` everywhere
  else so reports are byte-for-byte reproducible.
- **Determinism:** sort every string-keyed map before marshaling/rendering. Golden tests fail
  loudly on unsorted output.
- **Dependency injection:** `Logger`, `Transport`, `Clock`, `Runner`, operator I/O are injected
  via `Config`/`Deps`; `nil` falls back to a sensible default.
- **`model` purity:** keep `internal/model` stdlib-only and behavior-free. A reflection test
  forbids raw `time.Duration` fields in `Report`.

## Testing

Tests are **fully hermetic**. `go test ./...` needs no network, no external tools, no real
time. Follow these patterns when adding tests:

- **Fake external tools** with `runnertest.FakeRunner`: scripts match by
  `ArgsExact`/`ArgsPrefix`/`ArgsContain`, load canned output via `StdoutFile` from
  `testdata/{ip,ethtool,ping,iperf}/`, and synchronize via `Started`/`Delay` channels.
- **Never sleep to synchronize.** Use `Script.Started`/`Delay`, `FakeClock.Advance` (always
  after `BlockUntilWaiters(n)` — advancing before the code parks hangs forever), or
  `testutil.WaitFor`. Derive timeouts from `testutil.TestTimeout(t)`, never hardcode (`-race`
  runs 5–20× slower).
- **Golden files** (`testdata/golden/`) are byte-exact. Regenerate only via
  `go test ./internal/reporting -update`; `.gitattributes` keeps `testdata/**` binary so
  editors can't mangle line endings.
- **Leak checks:** call `testutil.LeakCheck(t)` in any test that spawns goroutines / an
  integration app.
- **Integration tests** (`internal/app`, white-box) use a **real** `Clock` (a shared
  `FakeClock` across two in-process apps deadlocks) with short intervals; each scenario gets
  its own `FakeRunner`.
- **Off-goroutine assertions** must use `t.Errorf` + return, never `t.Fatalf` (fatal off the
  test goroutine silently breaks the run).
- Config parsing uses a fresh `flag.NewFlagSet` per call — never `flag.CommandLine` (double
  parse panics).

## Security & process invariants

These are correctness requirements, not style preferences. A change that violates one is a bug.

- **Token:** lives only in `Config.Token` (compared constant-time via `protocol.TokenEqual`)
  and the PC1 startup banner (`fmt`, stdout). `model.Report` has **no token field by
  construction**. If you add a report field or a log line, confirm it can't carry the token.
- **Process termination:** never blanket `pkill`/`killall`. Verify ownership against
  `/proc/<pid>/{stat,cmdline}` (start-time, pgrp, argv0 basename) and `os.Geteuid()` before
  signaling. Kill by **process group** (`syscall.Kill(-pgid, …)`), SIGTERM grace then SIGKILL,
  and ignore `ESRCH`.
- **Run-state directory** trust boundary: prefer `$XDG_RUNTIME_DIR`, then root-only
  `/run/cablecheck`, `/tmp` last; reject symlinks and foreign-owned dirs (mode `0700`).
- **Frame decoding:** validate the 4-byte length prefix against the max **before** allocating
  (`TestFrameMaxSize` guards against a 4 GiB alloc). Keep it.
- **Report transfer:** only the three fixed filenames, size + SHA-256 verified, per-file/total
  caps; transfer failure is advisory and must never change the verdict or exit code.

### Exit-code contract (`internal/app/exit.go`)

`0` EXCELLENT/GOOD · `1` WARNING · `2` POOR/FAILED · `3` INCONCLUSIVE · `4` config, flag,
token/handshake, preflight & `doctor` failure · `5` peer/orchestration (abort, disconnect,
timeout, plan failure) · `6` local interrupt/SIGINT · `7` internal (e.g. report write).
Health classes are exactly: `EXCELLENT`, `GOOD`, `WARNING`, `POOR`, `FAILED`, `INCONCLUSIVE`.

## Git & commits

- **Claude owns git history.** Codex and other tools never commit or push (see below).
- **No AI attribution ever** — no `Co-Authored-By`, no "🤖 generated" lines, no mention of an
  assistant in commit messages, code, or docs.
- Commit messages: conventional, single-line scope (`fix: enforce transfer size bounds`,
  `feat: opt-in cable-test and TDR`). One logical change per commit.
- Only commit/push when the human asks. `.claude/`, `dist/`, the `cablecheck` binary,
  `cablecheck-report-*/`, and `*.test`/`*.out` are gitignored — do not force-add them.

## Using Codex as a second engine

The OpenAI Codex CLI (`codex`, config `~/.codex/config.toml`) is an independent model family.
Use it for **cross-model review** (it catches bugs Claude misses) and for **offloading
self-contained work** (separate account, so it doesn't consume Claude's usage limit).

```sh
# Read-only review / bug hunt — the default. Findings written to a durable file.
codex exec --cd /home/sakamoto/Projects/repos/cablecheck --sandbox read-only \
  --output-last-message .claude/workflows/codex-findings.md "<prompt>"

# Delegate an actual edit only when explicitly offloading a self-contained task:
codex exec --cd <dir> --sandbox workspace-write --output-last-message result.md "<prompt>"
```

Rules:

- **Codex never commits or pushes.** Claude reads its output, verifies it, then commits.
  This keeps every landed change human-reviewed and history AI-free.
- **Treat findings as a reviewer's, not directives.** Verify before acting, and dismiss false
  positives explicitly. A finding confirmed independently by 2+ sources is highest-confidence.
- When other agents are editing the tree, give Codex a **stable snapshot**:
  `git worktree add --detach /tmp/cc-snapshot HEAD` (remove with `git worktree remove` after).
- **Cross-review gated-fix loop** — the pattern that has caught real bugs here: gather findings
  from Codex + Claude review in parallel → triage into a durable findings doc under
  `.claude/workflows/` → fix one file-cluster at a time (test-first, each gated by
  `make check`) → Claude commits each fix → Codex adversarially re-verifies for regressions →
  loop until sound. Run the `-race -shuffle` gate **yourself** each round.

## Subagent economy (Claude)

Default spawned subagents to **Opus**. Reserve higher-cost judgment models for the
highest-leverage slots only: concurrency-heavy or security-sensitive review. Codex is
effectively a free third engine for adversarial verification.
